### 教育 × IT × 自動化ポートフォリオ  

### 🎯 このプロジェクトの目的
このリポジトリは、InDesign制作現場で発生する各記事向けの個別処理（初期整形）が目的です。
10年以上にわたり教育教材のデジタル化（デジタル教科書・ICT教材）に携わる中で、  
現場の負担を軽減し、品質と速度を両立するための仕組みを実装してきました。

---
### InDesign内の正規表現、検索置換（ページ移動・対象選択）
### スタイル設定（段落・文字）、16種類を一括処理するためパネルUI
---
### 全体の流れ
1. スクリプトを実行するとスクリプトパネルのダイアログが出て、
どのコーナー（A〜P）を処理するかチェックボックスで選ぶ
2. 「除外するもの」を自動判定して保護
• ロックされているフレーム・レイヤー
• マスターページ上のフレーム
• いま編集中のフレーム（カーソルが入っているところ）
3. 選んだコーナーごとの処理を実行
4. 結果レポートのダイアログメッセージ
• 何個のフレームを除外したか
• 各コーナーの処理結果
• メッセージ

### 🛠 実装されている代表機能
機能分割：16種類の記事処理を個別関数で管理（本来はあと共通処理１種類を足しています）
UI設計が丁寧に：ダイアログの視認性、操作性に配慮（フォントの大きさを変えられないため）
確認ダイアログの適切な配置：重要な処理前に確認
除外機能の実装：ロック・マスター・編集中フレームの自動除外は実務的（作業者の要望）
処理結果の詳細報告：何がどれだけ処理されたか明確
